#+TITLE: fnm.el - Emacs integration for Fast Node Manager
#+STARTUP: indent

* Introduction
fnm.el configures Emacs to use Node.js installations managed by
[[https://github.com/Schniz/fnm][Fast Node Manager (FNM)]].  It updates =PATH=, =exec-path=, and
=NODE_PATH= without evaluating shell output.

Version resolution is delegated to =fnm exec=.  FNM recursively resolves
=.nvmrc=, =.node-version=, and =package.json#engines.node=, then falls back to
its =default= alias.

* Requirements
- Emacs 28.1 or later
- A current =fnm= release with =exec= and =--resolve-engines=, available on
  =exec-path=

The package has no third-party Emacs Lisp dependencies.

* Installation
** With use-package and straight.el
#+begin_src emacs-lisp
  (use-package fnm
    :straight (:host github :repo "bobrowadam/fnm.el"))
#+end_src

** Using package.el
The package is not currently available from MELPA.

* Usage
** Select the project or default version
=M-x fnm-use= asks FNM to resolve the current directory and falls back to its
=default= alias.

#+begin_src emacs-lisp
  (fnm-use)
#+end_src

Pass any selector accepted by FNM to select it explicitly:

#+begin_src emacs-lisp
  (fnm-use "20")
  (fnm-use "lts-latest")
#+end_src

=M-x fnm-use-version= provides completion over installed semantic versions.

** Switch automatically
Enable the global minor mode to ask FNM to resolve Node when opening files in a
different directory:

#+begin_src emacs-lisp
  (fnm-auto-use-mode 1)
#+end_src

Disable it with =(fnm-auto-use-mode -1)=.  Run =M-x fnm-use= after changing a
version file in the current directory.

* Configuration
Set =fnm-executable= when =fnm= is not discoverable on =exec-path=.

* Development
Run the ERT suite with:

#+begin_src shell
  emacs -Q --batch -L . -l fnm-test.el -f ert-run-tests-batch-and-exit
#+end_src
